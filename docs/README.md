# Documentation Map

Optio keeps process and planning documentation separated from product runtime docs.

## Process and governance

- `docs/process/*` reusable delivery workflow and operating rules.
- `docs/testing/*` test and validation policy.

## Planning artifacts

- `docs/roadmap/*` backlog and iteration artifacts.
- `docs/features/*` feature-level design and rollout specs.

## Priority of truth

1. `AGENTS.md` and `docs/process/*` define mandatory workflow behavior.
2. `docs/roadmap/*` tracks execution status and planning commitments.
3. `docs/features/*` defines feature-level intent and acceptance.
4. `README.md` and `CLAUDE.md` remain the primary product and architecture references.
