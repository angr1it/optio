# Global Invariants

- Every meaningful change should map to at least one `BLG-###` item.
- Non-trivial work should have a feature spec.
- Iteration folders must keep `iteration.md`, `execution-plan.md`, and `status-report.md` synchronized.
- Repository documentation, code comments, and docstrings stay in English.
- Actionable execution steps live in `execution-plan.md` as Markdown checkboxes.
- Validation (`make check`) is required before declaring closure.
- Product-specific architecture and operations live in `README.md` and `CLAUDE.md`; planning docs should link to them instead of duplicating them.
